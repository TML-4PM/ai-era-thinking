# T4H Product Map v1

## Purpose
Turn the Signal / Place / Presence conversation into an executable product map, not a loose chat archive.

## Core Platform
- Synal Place: cross-surface context and presence layer.
- MyNeuralSignal: signal engine and state interpretation layer.
- ConsentX: consent, capture, sharing, persistence, and ambient-boundary governance.
- Reality Ledger: proof of what was captured, inferred, acted on, and shipped.

## Tier 1: Build Now
1. Signal Strip
   - TV, browser, mobile.
   - Ambient speedometer / traffic-light strip.
   - Shows state, not long explanations.
   - Max five dials: Momentum, Confidence, Pressure, Moment Potential, Insight Available.

2. Sport Companion: StatChat / StatThat
   - First wedge.
   - Watch with a context-aware companion.
   - User asks: what just happened, who is that, compare, is he good, why was that a penalty.
   - Works with live sport, phone, browser, and TV surface.

3. Context Companion
   - Universal three-command MVP:
     - What just happened?
     - Who / what is that?
     - Compare / is that good?
   - Works across TV, browser, event, work, learning, kitchen, garage.

4. Session Memory + Output
   - Match summary.
   - Watch-session recap.
   - Question history.
   - Follow-up actions.
   - Carry context across TV, phone, browser, event.

## Tier 2: Immediate Expansion
5. Mate Mode
   - Shared presence, not raw chat.
   - Reactions, group presence, aggregated sentiment, optional short messages.

6. Event Output Engine
   - Weddings, conferences, galas, sport events, concerts.
   - Capture → enhance → share → screen → reel/album/output before people leave.

7. Execution Bridge
   - Comet-style browser actions without building a browser.
   - Presence and context decide; execution tools act.
   - Supports Playwright, browser agents, APIs, Comet-like workflows.

## Tier 3: Strategic Builds
8. Mobility Companion
   - Car / commute / transport.
   - Telemetry, stress, time-to-destination, route confidence, calm prompts.

9. Care Mode
   - Elderly / accessibility.
   - Low-friction ambient reassurance, passive status, lightweight prompts.

10. OEM Adapter Layer
   - Siri, Samsung, LG, Android TV, Apple ecosystem, telcos, routers, retailers.
   - Integrate for distribution, not identity.

## Rule
No feature should be single-surface only. Every feature must have at least one path across TV, phone, browser, event, home, or mobility.

## Strategic Position
OEMs own devices. Browsers own pages. We own the moment, signal, continuity, and output.
